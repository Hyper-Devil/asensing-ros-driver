# 导远 INS5711DAA BDDB0B 协议解析规范

## 1. 协议概述

* **帧头 (Header)**：`0xBD 0xDB 0x0B` (共 3 字节)
* **整包长度 (Length)**：88 字节 (0 ~ 87 偏移)
* **字节序 (Endianness)**：小端模式 (Little-Endian)，低字节在前，高字节在后
* **校验方式 (Checksum)**：整包异或校验（XOR），多段校验位

---

## 3. INS 状态位说明 (偏移 39)

偏移 `39` 的字节通过按位与 (Bitwise AND) 判断组合导航收敛状态：

| 位数 (Bit) | 掩码 (Hex) | 物理意义 | 说明 |
| --- | --- | --- | --- |
| `bit 0` | `0x01` | 位置初始化 | `1`=已初始化，`0`=未初始化 |
| `bit 1` | `0x02` | 速度初始化 | `1`=已初始化，`0`=未初始化 |
| `bit 2` | `0x04` | 姿态初始化 | `1`=已初始化，`0`=未初始化 |
| `bit 3` | `0x08` | 航向初始化 | `1`=已初始化，`0`=未初始化 |

## 4. 轮询数据解析 (偏移 46~51, 56)

[cite_start]设备会利用 `Data1` (偏移46), `Data2` (偏移48), `Data3` (偏移50) 三个字段，通过偏移 `56` 的 `Type` 字节进行时分复用（轮询发送不同类型的数据）。

| Type (偏移 56) | 数据类型含义 | Data1 解析 | Data2 解析 | Data3 解析 |
| :---: | :--- | :--- | :--- | :--- |
| **`0`** | **定位信息精度** | 纬度 Std: `exp(Data1/100.0)` | 经度 Std: `exp(Data2/100.0)` | 海拔高度 Std: `exp(Data3/100.0)` |
| **`1`** | **定速信息精度** | 北向速度 Std: `exp(Data1/100.0)` | 东向速度 Std: `exp(Data2/100.0)` | 地向速度 Std: `exp(Data3/100.0)` |
| **`2`** | **姿态信息精度** | 横滚 Std: `exp(Data1/100.0)` | 俯仰 Std: `exp(Data2/100.0)` | 航向 Std: `exp(Data3/100.0)` |
| **`22`** | **设备内部温度** | 温度: `Data1 * 200.0 / 32768.0` (℃) | - | - |
| **`32`** | **GPS 状态** | GPS 定位状态 (详见下表) | 收星数 (个) | GPS 定向状态 (详见下表) |
| **`33`** | **轮速状态** | - | 轮速有无标志 (0=无, 非0=有) | - |

### 差分标志位状态表 (Type=32 时，Data1 与 Data3 的数值枚举)

[cite_start]当解析 GPS 定位状态或定向状态时，具体的数值对应以下释义 ：

* `0`: NONE (无解)
* `1`: FIXEDPOS (位置由FIX POSITION命令制定)
* `2`: FIXEDHEIGHT (暂不支持)
* `8`: DOPPLER_VELOCITY (速度由即时多普勒信息导出)
* `16`: SINGLE (单点定位)
* `17`: PSRDIFF (伪距差分定位)
* `18`: SBAS (SBAS定位)
* `32`: L1_FLOAT (L1浮动解)
* `33`: IONOFREE_FLOAT (消电离层浮点解)
* `34`: NARROW_FLOAT (窄巷浮点解)
* `48`: L1_INT (L1固定解)
* `49`: WIDE_INT (宽巷固定解)
* `50`: NARROW_INT (窄巷固定解)

## 5. 校验代码参考

对于整包解析，需要对第 `0` 到 `86` 字节依次进行异或操作，并与第 `87` 字节（即 `Length - 1`）比对：

```cpp
char xorcheck = 0;
// 包长 Length = 88
for (int i = 0; i < Length - 1; i++) {
    xorcheck = xorcheck ^ input[data_packet_start + i];
}
if (input[data_packet_start + Length - 1] == xorcheck) {
    // 校验通过，开始提取数据
}

```
